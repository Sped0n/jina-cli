# Plan: global timeout, refined retry policy, and local-feature removal

## Recommended approach

Implement the new behavior by keeping all HTTP policy changes centralized in `jina_cli/api.py`, propagating a single global `--timeout` value from the root Click group in `jina_cli/main.py`, and simplifying the codebase by removing all local-mode code paths and the `grep` subcommand entirely.

This keeps the change small and coherent:
- one global timeout source in CLI context
- one shared retry policy in the HTTP request helper
- fewer special cases after removing localhost/local-embedding flows

## Files to modify

- `jina_cli/main.py`
- `jina_cli/api.py`
- `tests/test_cli.py`
- `tests/test_local.py`
- `skills/jina-cli/SKILL.md`
- `skills/jina-cli/references/basics.md`
- `skills/jina-cli/references/search-read.md`
- `skills/jina-cli/references/vectors-ranking.md`
- `skills/jina-cli/references/pipes-scripting.md`

## Implementation steps

1. Add a global timeout flag in `jina_cli/main.py`.
- Add root `@click.option("--timeout", type=float, default=None, help=...)` on `cli()`.
- Store it in `ctx.obj["timeout"]` alongside `api_key`.
- Update every remaining HTTP-backed subcommand to read `timeout = ctx.obj.get("timeout")` and pass it into the corresponding API function.

2. Remove local-mode CLI support from `jina_cli/main.py`.
- Delete `--local` options and all local-mode branching from `embed`, `rerank`, `dedup`, and `classify`.
- Clean up command help text and examples so they only describe remote behavior.
- Remove the `grep` subcommand completely.
- Remove `grep` and local-related entries from top-level usage/help output.

3. Centralize timeout propagation in `jina_cli/api.py`.
- Extend all remaining HTTP-backed public API functions to accept `timeout: float | None = None`.
- Resolve the effective timeout inside each function as `timeout or DEFAULT_TIMEOUT` when creating `_client(...)`.
- Keep existing command-specific longer defaults only where still intended; if the user’s global `--timeout` is set, it should override those built-in defaults.
- Ensure helper calls such as `search_blog()` pass timeout through to `search_web()`.

4. Refine retry behavior in `jina_cli/api.py`.
- Keep `_request_with_retry()` as the single retry implementation point.
- Update retry backoff to `0.5`, `1.0`, `2.0` seconds.
- Keep retry scope limited to transient network errors plus HTTP `429` and `5xx` from Jina-owned endpoints.
- Do not retry `4xx` other than `429`.
- Preserve `Retry-After` support for `429`, but cap sleep to 30 seconds.
- Add a small helper or inline logic so the request helper only retries `5xx` for Jina endpoints (`api.jina.ai`, `r.jina.ai`, `svip.jina.ai`), which satisfies the requirement to retry Jina service failures without retrying target-site issues.

5. Remove local helper code from `jina_cli/api.py`.
- Delete `LOCAL_SERVER` and the local helper functions (`local_embed`, `local_classify`, `local_rerank`, `local_deduplicate`).
- Keep shared math/helpers only if still used after removal; otherwise delete dead code.

6. Update tests.
- Replace `tests/test_local.py` with retry/timeout-focused unit tests for `jina_cli.api` rather than localhost behavior.
- Add unit coverage for:
  - retry on `429`
  - retry on Jina `503`/`500`
  - no retry on `404`
  - no retry on non-Jina endpoints returning `5xx`
  - retry on `httpx.TimeoutException` / `httpx.ConnectError`
  - `Retry-After` handling capped at 30 seconds
- In `tests/test_cli.py`, add CLI coverage that `--timeout` is accepted globally and works before subcommands, for example `jina --timeout 45 search ...` and `jina --timeout 60 read ...`.
- Remove or rewrite tests that reference local-mode functionality.

7. Update skill/docs content to match the simplified CLI.
- Remove references to `--local`, local embedding server workflows, and `jina grep` from the skill and reference docs.
- Add one short guidance/example note for the new global `--timeout` flag in `skills/jina-cli/references/basics.md`.
- Add one example in `skills/jina-cli/references/search-read.md` showing `jina read ... --timeout ...` or `jina --timeout ... read ...`.
- Update vector/ranking docs so they describe only remote commands.
- Update pipes/scripting docs if they mention old local or grep workflows.

## Key design notes

- The timeout flag should be global and apply to all remaining HTTP-backed commands, not just `read` and `search`.
- After removing local-mode code, there is no need to define special localhost retry semantics.
- The cleanest way to satisfy “retry Jina 5xx but not target site issues” is to classify retryability by the requested endpoint host inside `_request_with_retry()`; all Jina-owned API hosts are known centrally.
- This is a minimal-change path because the CLI already uses `ctx.obj` and the API layer already routes requests through one shared helper.
